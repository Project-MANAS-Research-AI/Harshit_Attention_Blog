Understanding Attention: A Beginner-Friendly Guide

This blog post explains the attention mechanism introduced in "Attention Is All You Need" (Vaswani et al., 2017).

Why Attention?

Traditional RNNs process information sequentially. As sequences become longer, remembering important information becomes difficult. Attention solves this by allowing every token to directly interact with every other token.

Query, Key and Value

Imagine searching for a book in a library.

  • Query = what you are looking for
  • Key = label of each book
  • Value = information inside the book

The attention mechanism compares the query against all keys and retrieves the most relevant values.

Scaled Dot Product Attention

Attention is computed using:

Attention(Q,K,V) = softmax(QKᵀ / √dₖ)V

Why divide by √dₖ?

Without scaling, dot products become very large for high-dimensional vectors. The softmax then becomes extremely peaked and gradients become unstable. Scaling keeps training stable.

Multi-Head Attention

Instead of using a single attention mechanism, Transformers use multiple attention heads.

Different heads can focus on different relationships:

  • Object relationships
  • Long-range dependencies
  • Syntax
  • Semantic meaning

Positional Encoding

Transformers have no recurrence. They process all tokens simultaneously. Positional encodings inject information about token order.

Connection to Vision Transformers

In NLP, tokens represent words. In Vision Transformers, tokens represent image patches. The same attention mechanism is reused almost unchanged.

Conclusion

Attention enables direct interactions between tokens, making it possible to model long-range dependencies efficiently. This idea became the foundation of modern language models and Vision Transformers.
