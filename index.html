Understanding Attention: A Beginner-Friendly Guide

Attention is one of the most influential ideas in modern deep learning. It powers large language models such as ChatGPT, Gemini and Claude, and it is also used in Vision Transformers for computer vision tasks. This blog explains the core idea behind attention and develops an intuition for Query, Key and Value vectors without assuming advanced mathematical knowledge.

Why Was Attention Needed?

Before Transformers, sequence modeling was dominated by recurrent neural networks (RNNs) and LSTMs. These models process information one token at a time. Although effective, they often struggle to capture long-range dependencies.

For example, consider the sentence:

"The cat sat on the mat because it was tired."

To understand the meaning of "it", the model must connect it back to "cat". Attention allows every word to directly interact with every other word, making these relationships easier to learn.

The Intuition Behind Query, Key and Value

The concepts of Query, Key and Value (Q, K, V) are central to attention. A useful analogy is a library.

  • Query: What information am I looking for?
  • Key: What information do I contain?
  • Value: What information should I provide if selected?

Each token in a sentence is transformed into three vectors: a Query vector, a Key vector and a Value vector. The Query is compared against all Keys. The strongest matches determine which Values contribute most to the output.

Where Do Q, K and V Come From?

Each token is first converted into an embedding vector. The Transformer then applies three learned linear transformations:

  • One projection generates Queries.
  • One projection generates Keys.
  • One projection generates Values.

Although all three originate from the same embedding, they learn different roles during training.

Scaled Dot-Product Attention

The attention mechanism is computed using:

Attention(Q,K,V) = softmax(QKT/√dk)V

Step 1: Compute Similarities

The dot product between Queries and Keys measures similarity. A larger value indicates a stronger relationship.

Step 2: Apply Softmax

Softmax converts similarity scores into probabilities that sum to one. This determines how much attention should be assigned to each token.

Step 3: Combine Values

The attention weights are used to compute a weighted average of Value vectors. The result becomes the new representation of the token.

Why Divide by √dk?

As vector dimensionality increases, dot products naturally become larger. Large values can cause the softmax function to become extremely peaked, which leads to unstable gradients during training.

Dividing by √dk normalizes the scale of the scores, keeping training stable and improving optimization.

Multi-Head Attention

Instead of using a single attention operation, Transformers use multiple attention heads in parallel.

Each head learns a different type of relationship.

  • One head may focus on nearby words.
  • Another may capture grammatical structure.
  • Another may identify semantic relationships.
  • Another may focus on long-range dependencies.

The outputs of all heads are concatenated and combined, allowing the model to capture richer information than a single attention mechanism.

Positional Encoding

Unlike RNNs, Transformers process all tokens simultaneously. This means they have no built-in understanding of word order.

To solve this problem, positional encodings are added to token embeddings. The original Transformer uses sinusoidal functions with different frequencies, allowing the model to encode both absolute and relative positions.

Without positional encodings, the sentence:

"Dog bites man"

would look identical to:

"Man bites dog"

even though the meanings are completely different.

Attention in Vision Transformers

The same attention mechanism used in NLP can also be applied to images. Vision Transformers divide an image into small patches. Each patch is treated as a token.

Self-attention then determines which image regions are important for classification. For example, when recognizing a bird, the model may focus on the beak, wings and body while ignoring much of the background.

This reuse of attention is one reason Transformers have become successful across many domains.

Key Takeaways

  • Attention allows every token to interact with every other token.
  • Queries search for relevant information.
  • Keys describe available information.
  • Values contain the actual information used to build outputs.
  • Scaling by √dk stabilizes training.
  • Multi-head attention learns multiple relationships simultaneously.
  • Positional encoding provides information about token order.
  • The same mechanism is used in Vision Transformers.
